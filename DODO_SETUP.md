## 🐛 Troubleshooting

### Issue: "Verification service misconfigured"
**Cause**: Invalid or missing DODO_API_KEY

**Fix**:
1. Check Vercel environment variables
2. Verify API key is correct: `wrZkEG7eTHJvR5cE...`
3. Run `node test-dodo.js KEY` to test locally

### Issue: "Invalid license key"
**Cause**: License doesn't exist in Dodo

**Fix**:
1. Check if customer completed payment
2. Verify license was generated by Dodo
3. Check Dodo dashboard → Licenses
4. Run test script to see all licenses

### Issue: "Connection failed"
**Cause**: Network error or Vercel deployment issue

**Fix**:
1. Verify Vercel deployment is live
2. Check: https://aaaaaa-alpha-inky.vercel.app/api/verify
3. Check browser console for CORS errors

### Issue: Webhook not receiving events
**Cause**: Webhook not configured or URL incorrect

**Fix**:
1. Verify webhook URL in Dodo: `https://aaaaaa-alpha-inky.vercel.app/api/webhook`
2. Check Vercel logs for incoming webhook events
3. Test webhook from Dodo dashboard (send test event)

## 📊 How It Works

```
Customer pays on Dodo checkout
         ↓
Dodo generates license key
         ↓
Dodo sends webhook to Vercel → /api/webhook
         ↓
Customer receives key via email
         ↓
Customer enters key in extension
         ↓
Extension calls → /api/verify
         ↓
Vercel checks Dodo API → GET /v1/licenses/{key}
         ↓
Vercel returns { pro: true } to extension
         ↓
Extension stores license locally → Pro unlocked!
```

## 🔒 Security Notes

1. **Never expose API keys in client-side code**
   - ✅ API key is only in Vercel environment variables
   - ✅ Extension calls Vercel API, not Dodo directly
   - ✅ Webhook secret verifies incoming events

2. **Webhook signature verification**
   - The webhook handler is set up to verify signatures
   - In production, implement HMAC verification for extra security

3. **License validation**
   - Licenses are validated server-side via Vercel
   - Extension only stores the result locally
   - No way to bypass verification from client-side
